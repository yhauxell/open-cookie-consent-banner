# OpenConsent Brand Identity

## Brand Name

**OpenConsent** (product name)  
**openconsent.dev** (domain/brand URL)

## Brand Positioning

### Core Identity

**"The open-source consent management platform built for modern developers."**

### Brand Pillars

1. **Open** - Transparent, open-source, community-driven
2. **Developer-First** - Built by devs, for devs
3. **Modern** - shadcn/ui native, TypeScript-first, React 19+
4. **Compliant** - GDPR, CCPA, Consent Mode v2 ready

## Taglines & Messaging

### Primary Tagline

**"Consent management, open and simple."**

### Alternative Taglines

- "Open consent for modern apps"
- "GDPR compliance, open source"
- "The consent platform developers actually want to use"
- "One command. Full compliance. Zero vendor lock-in."

### Value Propositions

#### For Developers

- ✅ **One command install** via shadcn CLI
- ✅ **TypeScript-first** with full type safety
- ✅ **Zero dependencies** beyond shadcn/ui
- ✅ **Self-hosted** - no SaaS subscription
- ✅ **Fully customizable** - it's your code

#### For Businesses

- ✅ **GDPR & CCPA compliant** out of the box
- ✅ **Google Consent Mode v2** ready
- ✅ **Full audit trail** for compliance
- ✅ **Free forever** - no per-domain pricing
- ✅ **No vendor lock-in** - own your data

## Brand Voice

### Tone

- **Professional but approachable** - Not corporate, not too casual
- **Technical but clear** - Respects developer intelligence
- **Confident but humble** - We're good, but we're open to feedback
- **Helpful** - We solve real problems

### Writing Style

- Use "you" (direct address)
- Short sentences
- Code examples over explanations
- Show, don't tell
- Avoid marketing fluff

## Feature Naming

### Core Features

- **OpenConsent Provider** (not "CookieConsentProvider")
- **OpenConsent Banner** (not "CookieBanner")
- **OpenConsent Settings** (not "CookieSettings")
- **OpenConsent Script** (not "ConsentScript")

- **OpenConsent Mode** (Google Consent Mode v2 integration)
- **OpenConsent Trace** (audit trail feature)
